GitHub Pages 部署指南

如何将个人网站部署到GitHub Pages并公开访问

什么是GitHub Pages？

GitHub Pages是GitHub提供的一项免费服务，允许您直接从GitHub仓库托管静态网站。它是部署个人网站、项目文档或作品集的理想选择。

部署成功后，您的网站将可以通过以下URL访问：

https://您的用户名.github.io

例如：如果您的用户名是"liyitang"，则网站地址为 https://liyitang.github.io

详细部署指南

1
创建GitHub账户

访问 GitHub官网 并注册账户。

如果您已有账户，请直接登录。

2
创建新仓库

登录后，点击右上角的"+"号，选择"New repository"。

重要： 仓库名称必须为 您的用户名.github.io

例如，如果您的用户名是"liyitang"，则仓库名应为"liyitang.github.io"。

选择"Public"（公开），并勾选"Add a README file"。

提示： 使用用户名.github.io的命名约定，您的网站将直接通过https://用户名.github.io访问，无需额外设置。
3
上传HTML文件

在仓库页面，点击"Add file" → "Upload files"。

将您的HTML文件（例如index.html）拖拽到上传区域。

添加提交信息（如"Initial commit"），然后点击"Commit changes"。

注意： 确保您的主页文件名为index.html，这是GitHub Pages默认寻找的主页文件。
4
启用GitHub Pages

进入仓库的"Settings"选项卡。

在左侧菜单中找到"Pages"。

在"Source"部分，选择"Deploy from a branch"。

选择"main"分支和"/ (root)"文件夹，然后点击"Save"。

完成！几分钟后，您的网站将在 https://您的用户名.github.io 上线。

常见问题解答

我的网站没有立即显示，怎么办？

GitHub Pages部署通常需要几分钟时间。如果您的网站在10-15分钟后仍然无法访问，请检查：

  • 仓库名称是否正确（必须是用户名.github.io）
  • 是否已启用GitHub Pages（在Settings > Pages中查看）
  • 是否已将index.html文件上传到仓库的根目录
我可以使用自定义域名吗？

是的，您可以将自定义域名指向GitHub Pages网站。在仓库的Settings > Pages中，找到"Custom domain"部分，输入您的域名，然后按照说明配置DNS记录。

如何更新我的网站？

只需更新您的HTML文件，然后将更改提交到GitHub仓库。GitHub Pages会自动重新部署您的网站，通常几分钟内更新就会生效。

GitHub Pages支持哪些类型的网站？

GitHub Pages主要支持静态网站（HTML、CSS、JavaScript）。它不支持服务器端语言如PHP或数据库。对于动态内容，您可以考虑使用Jekyll（GitHub Pages内置支持）或其他静态站点生成器。
